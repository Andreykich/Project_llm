# Рубрика оценки качества (human / LLM-judge)

Оценивание проводится по 4 шкалам 0–5 (целые значения). Рекомендуется 2 оценщика.

## 1) Relevance — соответствие запросу (0–5)
0: ответ не про заданную сферу/ограничения  
3: в целом по теме, но есть важные промахи  
5: точно соответствует сфере, ограничения учтены (город/бюджет/формат/ЦА)

## 2) Specificity — конкретика (0–5)
0: общие слова  
3: есть частичная конкретика (1–2 раздела)  
5: конкретные сегменты, каналы, примеры, шаги, цифры/диапазоны

## 3) Actionability — практичность (0–5)
0: нельзя применить  
3: есть общий план, но без действий/приоритетов  
5: есть понятный MVP/план (>=5 шагов), эксперименты, риски+митигейшн

## 4) Evidence — обоснованность источниками (0–5)
0: нет ссылок/источников  
3: есть источники, но слабая связь с тезисами  
5: ключевые тезисы подкреплены релевантными источниками (>=2)

## Итоговый балл
`quality_mean = mean(4 шкалы)`

## Минимальный порог PASS (для pass-rate)
PASS если:
- relevance >= 3
- actionability >= 3
- evidence >= 2
